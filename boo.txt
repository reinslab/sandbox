git flow でfeatureブランチreleaseブランチをstartすると
ブランチはdevelopブランチから作成される。
hotfixブランチをstartすると
ブランチはmasterブランチから作成される。

リモート側　origin/develop をチェックアウトすると
HEAD detached at origin/develop
となり、リモート側のファイルの状態を確認できる。

git checkout develop するとHEAD detached は解除される。

detached とは「分離した」「孤立した」という意味で

普通のHEAD：ブランチを指している　HEAD → master → コミット
detached HEAD：コミットを指している　HEAD → コミット

HEADは「今いるブランチの最新のコミット」だから
detached HEAD は「ブランチから分離したコミット」

git commit -am "update"

でまとめてaddとcommitができる。
まとめて行うコマンドの場合
git add xxx.html はOKだが
git commit xxx.html はerrorとなる。
